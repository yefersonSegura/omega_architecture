Vision

Intent-first collaboration — flows orchestrate, agents handle IO, one shared channel. Pairs naturally with the Angular package omega-angular for full-stack teams.

OmegaChannel & events

Broadcast typed events, namespaces for modules, and optional contracts in debug — see the channel guide.

Flows & agents

OmegaFlow receives intents and events; OmegaAgent runs behaviors. The UI stays thin; business rules live outside widgets.

Tooling

omega CLI (create app, generate ecosystem, traces, optional Omi AI), inspector overlay or browser + VM Service, and time-travel traces.

What is this site? ​

This site documents the omega_architecture Dart package and the omega_architecture GitHub repository — Flutter only.

  • Read Vision & why Omega for purpose and fit, then Getting started and Core concepts.
  • Follow Data flow for an end-to-end picture, then Channel & events, Intents & flows, and Agents & behaviors.
  • Use API reference as the map to the generated API docs on pub.dev.

Long-form Markdown in the repo lives under doc/ (e.g. GUIA.md). The online Inspector page (VM Service) ships as a static asset next to this site.

Monorepo layout, example/, and publishing are summarized under Repository layout and the README on GitHub.
